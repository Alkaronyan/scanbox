2. **Component Separation**:

   * Vid_Mux_TEST Container: Mock camera scaffold. Compiles v4l2loopback against host kernel, creates /dev/video200, feeds SMPTE synthetic stream.
   * scanbox_dhcp Container: DHCP server for USB NCM link. Runs dnsmasq on host network, assigns IPs 192.168.55.100-200 to Windows PC.
3. **Deterministic Device Mapping**:
   * Input 0 (Physical): /dev/video100 ← /dev/v4l/by-id/usb-046d_0809_5DD0F8C2-video-index0 (Logitech C270)
   * Input 1 (Mock): /dev/video200 ← created by Vid_Mux_TEST via v4l2loopback video_nr=200

   * Output: MJPEG HTTP stream at /stream (port 5000) — accessible over WiFi/LAN or USB NCM direct link
4. **Code Standards**: discussions in Spanish or English, all code/comments/files in English.
5. **Startup orchestration**: docker-compose.yml is the single source of truth for all containers. scanbox-stack.service (systemd) runs `docker compose up -d` at boot, respecting depends_on ordering.

### Kernel & Environment
* Host kernel: 6.18.29+rpt-rpi-v8
* KBUILD_DIR: /usr/lib/linux-kbuild-6.18.29+rpt (stored in .env, regenerated by setup_host.sh on kernel update)
* v4l2loopback version: 0.15.3 (upgraded from 0.13.2 for kernel 6.18 compatibility)
  * MUST use `exclusive_caps=0` — v0.15.x defaults to exclusive_caps=1 which hides V4L2_CAP_VIDEO_CAPTURE
  * MUST use `io-mode=rw` on the v4l2src reading from video200 — v4l2loopback does not support GStreamer's MMAP buffer allocation
