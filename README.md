# SCANBOX вЂ” Dynamic Video Switcher Module

A deterministic video source switcher for Raspberry Pi 4. Hot-swaps between a physical USB camera and a synthetic mock source via a REST API and web UI. Accessible over LAN/WiFi or directly via USB cable (no router needed).

## Architecture

### Containerization (strict)
Everything that can run in a container does. The host OS stays minimal. Only Docker Engine, kernel headers, git, and systemd service files are installed on the host вЂ” all automated by `scripts/setup_host.sh`.

### Containers

| Container | Purpose |
|---|---|
| **Vid_Mux** | GStreamer pipeline, Flask REST API, MJPEG stream, Web UI. Port 80. |
| **Vid_Mux_TEST** | Compiles v4l2loopback v0.15.3 against the host kernel, creates /dev/video200, feeds a live SMPTE test pattern into it. |
| **scanbox_dhcp** | dnsmasq DHCP server on usb0. Assigns IP to the connected Windows PC automatically. |

### USB NCM Direct Link

| Side | IP |
|---|---|
| Pi | 192.168.55.1 |
| Windows PC | 192.168.55.100вЂ“200 (DHCP, automatic) |

Windows 11 recognises the Pi as a USB ethernet adapter natively вЂ” no drivers to install.

## Quick Start

### Fresh Pi setup (run once)
```bash
sudo ./scripts/setup_host.sh
sudo reboot
```
This installs Docker, kernel headers, systemd services, and generates `.env`. After reboot the full stack starts automatically.

### Manual start / rebuild after code changes
```bash
./scripts/rebuild_vid_mux.sh
```
Detects cameras dynamically, rebuilds the vid_mux image, and relaunches the full stack.

### Access the web UI
| Connection | URL |
|---|---|
| USB cable (direct) | http://192.168.55.1 |
| WiFi / LAN | http://\<pi-ip\> |

## Boot Sequence

```

    в”њв”Ђв”Ђ scanbox_dhcp     в†’  DHCP server on usb0
    в”њв”Ђв”Ђ vid_mux_test     в†’  compiles v4l2loopback в†’ /dev/video200 (healthcheck)
    в”‚    в””в”Ђв”Ђ waits for /dev/video200 to appear (polls every 2s, 120s timeout)
    в””в”Ђв”Ђ vid_mux          в†’  built with dynamically detected cameras в†’ Flask 
```

`scanbox.service` replaces the old `scanbox-stack.service`. Key advantage: cameras are detected dynamically at every boot, so plugging in a new camera is pick-up-and-go.

## REST API

| Method | Path | Description |
|---|---|---|
| GET | / | Web UI |
| GET | /stream | Live MJPEG stream |
| GET | /api/v1/status | Active source + full source list |
| POST | /api/v1/source | Switch source `{"source_id": 0\|1\|...}` |
| POST | /api/v1/snapshot | Capture frame to disk |
| GET | /api/v1/snapshot/last | Retrieve last snapshot |
| GET | /api/v1/camera/controls | V4L2 controls for the active source (empty for mock) |
| POST | /api/v1/camera/control | Set V4L2 control `{"control": "saturation", "value": 128}` |
